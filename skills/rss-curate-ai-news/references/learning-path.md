## 关注范围

Top K 的默认主线是帮助不预设机器学习知识的读者逐步理解 AI、LLM 和智能体，并把知识用于趋势判断、商业决策和项目推进。优先选择能推进以下至少一项的文章：

1. 建立 AI/机器学习基础概念或纠正常见误解；
2. 解释 LLM 的能力来源、限制、使用方法或评测证据；
3. 解释智能体的工具调用、任务编排、记忆、反馈和可靠性；
4. 把模型能力连接到产品工作流、工程约束、采用路径或商业经济性；
5. 提供可验证的趋势信号、项目动作、指标或停止条件。

计算机视觉、语音、机器人、芯片、数据中心、纯理论研究及其他相邻领域，只有同时满足以下条件才可进入 Top K：

- 与通用模型或 LLM/Agent 的能力、成本、算力基础设施、风险监管或价值链存在明确因果链；
- 文章提供主线可复用的概念，或会实质改变趋势、商业、产品或项目判断；
- 这种联系来自文章或上游证据，而不是仅因标题出现“AI”而推断。

不满足条件的内容仍进入全量审计，标为范围外，不参与 Top K。技术论文不因载体受罚，也不因“论文”“SOTA”或来源品牌获得优势；只评价其学习、证据和决策价值。

## 六段概念脊柱

| 阶段 | 核心问题 | 代表概念 | 通向下一段的能力 |
| --- | --- | --- | --- |
| 1. AI/ML 基础 | 模型如何从数据形成可泛化行为？ | 机器学习、训练/推理、数据集、神经网络、泛化、过拟合 | 区分训练、推理与规则程序 |
| 2. LLM 核心 | 文本模型如何表示、生成并受上下文约束？ | 大语言模型、Transformer、词元、嵌入、上下文窗口、预训练、幻觉 | 用“概率生成＋上下文条件”解释能力与边界 |
| 3. LLM 应用 | 如何让通用模型可靠利用业务知识完成任务？ | 提示工程、结构化输出、检索增强生成、微调、工具调用、评测 | 为具体任务选择增强方式并定义验证指标 |
| 4. Agent 系统 | 如何把模型组织成可执行、可反馈的多步系统？ | AI 智能体、工作流编排、规划、记忆、工具、状态、人机协同 | 分清模型能力、系统控制与外部执行 |
| 5. 工程可靠性 | 如何在真实环境控制质量、成本和风险？ | 可观测性、护栏、权限、延迟、吞吐量、回退、红队测试 | 用失败模式和运行指标决定是否上线或扩展 |
| 6. 产品与商业 | 技术何时转化为用户价值和可持续经济性？ | 场景适配、采用障碍、单位经济、构建或购买、价值链、竞争壁垒 | 把机制变化连接到运营指标、经济性和决策 |

这是一条依赖脊柱，不是固定课程表。文章可以跨段，但只指定一个主概念；用支撑概念表达跨段连接。

## 关键前置关系

按理解依赖而非学术完备性选择前置概念。使用下列主干关系；文章若能用更短路径自洽讲清，可以省略非必要节点。

```text
机器学习 → 训练与推理 → 大语言模型
神经网络 → Transformer → 大语言模型
词元 + 嵌入 + Transformer → 上下文窗口与生成
大语言模型 + 上下文窗口 → 提示工程与结构化输出
嵌入 + 检索 + 大语言模型 → 检索增强生成（RAG）
训练与推理 + 数据集 → 微调与模型评测
大语言模型 + 工具调用 + 工作流编排 → AI 智能体
AI 智能体 + 状态 → 记忆、规划与多步执行
模型评测 + 可观测性 + 护栏 → 工程可靠性
延迟 + 吞吐量 + 推理成本 → 单位经济
工程可靠性 + 场景适配 + 单位经济 → 构建、购买、观察或忽略
```

不要把上图之外的全部相关术语机械列为先修。只保留不理解就无法看懂主机制、完整例子或决策结论的概念。

## 概念标注契约

评分前为每个可评价审查单元记录：

- **一个主概念**：文章最值得读者带走、能在后续复用的单一概念；必须使用规范名。
- **零至两个支撑概念**：用于解释主概念或建立跨段连接；不得用多个近义词占位。
- **零至三个必要前置概念**：只记录理解文章所必需的概念，并按依赖顺序排列。
- **先修缺口**：`0 已覆盖`、`1 可用两分钟补齐`、`2 本卡无法补齐`。
- **学习动作**：`初识`、`加深`、`连接`、`纠偏`、`迁移` 中恰好一个。

先修缺口按以下规则判断：

- `0 已覆盖`：没有必要前置，或所有必要前置概念已在历史 `story_concepts` 中曝光，或由本批建议阅读顺序中更早的入选项覆盖。这里只表示接触过，不表示掌握。
- `1 可用两分钟补齐`：即使没有历史曝光，也能在学习卡内用定义、用途、边界和一个最小例子准确补齐。
- `2 本卡无法补齐`：理解核心主张需要较长推导、多个未曝光前置或专业训练，无法在不扭曲机制的前提下补齐。

学习动作描述文章实际支持的最高可靠学习操作；历史曝光用于判断是否产生跨期增量，不用于推断读者状态：

- `初识`：建立主概念的最低可用心智模型。主概念未历史曝光时通常构成首次曝光；已历史曝光却只重复同层定义时仍标 `初识`，并另写“重复曝光、无跨期增量”，不得包装成加深。
- `加深`：在已曝光概念上增加机制、约束、比较或证据层级。
- `连接`：明确说明两个或更多已曝光概念之间的因果、依赖或权衡。
- `纠偏`：文章有证据地纠正领域中的常见错误模型；不要声称读者本人持有该误解。
- `迁移`：把已曝光概念用于新的场景、产品、行业或决策问题，并说明适用边界。

若文章对已曝光主概念只重复同层定义或同类例子，学习动作字段保持 `初识`，学习阶梯与认知迁移最高记 8，且不得标为学习桥梁。该标注只说明文章停留在入门解释层，不表示读者尚未掌握或必须重学。

## 规范概念名与别名

概念标签使用“中文规范名”或“中文规范名（通用缩写）”。匹配别名时忽略大小写和全半角差异，但先按上下文消歧。公司、产品版本、论文名、人物、会议和一次性事件不是概念。

| 规范概念名 | 合并到该名称的常见别名 |
| --- | --- |
| 人工智能（AI） | 人工智能、Artificial Intelligence、AI |
| 机器学习（ML） | 机器学习、Machine Learning、ML |
| 神经网络 | Neural Network、NN、深度神经网络 |

| 大语言模型（LLM） | 大模型、语言大模型、Large Language Model、LLM |
| Transformer | Transformer 架构、Transformer Model |
| 词元 | Token、Tokens、分词单元 |
| 嵌入 | Embedding、Embeddings、向量表示 |
| 上下文窗口 | Context Window、上下文长度、Context Length |

## 解释层级

为主概念标记本篇最高可靠解释层级，并补齐读者到达该层所需的较低层内容：

- **L1 定义与用途**：它是什么、解决什么问题、一个非例，以及不能做什么。
- **L2 机制与约束**：输入、过程、输出、关键资源、失败条件和能力边界。
- **L3 比较与评测**：与替代方案的差异、评测设计、证据质量、权衡和不确定性。
- **L4 项目与商业判断**：场景、采用障碍、运营指标、单位经济、决策阈值和推翻信号。

层级表示本篇解释达到哪里，不表示读者能力等级。没有证据支撑时不得为了凑 L4 编造客户、市场规模、成本或投资结论。

## 历史曝光与跨期递进

`story_concepts` 只表示某个规范概念曾随已选审查单元曝光。先把别名归一为规范名；同一 `story_key` 下的同一概念只计一次曝光，同一 `story_key` 下的不同概念形成一次共现。可以据此：

- 按规范名统计曝光次数；
- 统计同一审查单元中的概念共现；
- 区分新概念与复现概念；
- 为复现概念优先选择加深、连接、纠偏或迁移，而非再次停留在同一层定义；
- 识别长期未出现的脊柱缺口，作为后续选稿的参考因素。

不得据此推断读者已经阅读、理解、记住或掌握概念，也不得生成掌握度、正确率或能力等级。曝光次数不直接加分；只有文章确实推进学习层级或形成新连接时才构成学习价值。共现只说明概念曾一起出现，不能证明二者存在因果关系。

## 单篇自洽与批次编排

每张学习卡必须能独立阅读：简短回顾已曝光的必要前置，为缺口 1 提供两分钟先修补丁，并用完整的输入→过程→输出例子解释主概念。不得要求读者返回历史报告才能理解核心结论。

跨期递进通过“减少重复铺垫、提高解释层级、增加概念连接和迁移场景”实现，而不是省略必要定义。复现概念至少说明本篇相较历史曝光新增了什么；若没有可识别增量，不把重复出现包装成学习进阶。

同一批次同时输出价值排名和建议阅读顺序。价值排名反映综合选择价值；建议阅读顺序先满足前置依赖，再从低难度到高难度排列。若后文依赖前文补齐先修，在相应条目中明确指出依赖；即使按价值排名单独打开任一学习卡，它仍须保持自洽。
