# 双层日报输出契约

正常完成时严格使用以下顺序。报告正文在 Codex 对话、本地 Markdown 和邮件正文中保持一致。

```markdown
# AI 例会日报｜YYYY-MM-DD

> 运行时间：YYYY-MM-DD HH:mm（Asia/Shanghai）
> 审查窗口：...
> 本次监控源：N 个“-今天看啥”订阅（逐一列名）＋橘鸦AI早报
> 源条目与拆分：窗口内 FreshRSS 源条目 D 条；早报 A 期，识别新闻 E 条，原文未解析 U 条；规范原文去重后共 C 个审查单元
> 内容分类：普通候选 N 条、商业关联 N 条、非选稿 N 条、原文未解析 N 条
> 今日判断：一句话说明整体质量和最重要的共同信号

> 回溯补位：否 / 是（原始发布日期）
```

## 覆盖与表达检查

- 排名矩阵和逐条短评覆盖去重后的每个审查单元；橘鸦父早报只出现在拆分覆盖和发现入口中，不得被评分或成为 Top 2。
- 同一规范原文由多个入口发现时只占一行，并列出全部入口；不同原文报道同一事件时各占一行但归入同一报道簇。
- 原文未解析单元不得静默删除。使用早报入口链接，写明未解析状态和证据缺口；不生成六维分数，也不参与 Top 2。
- 回溯单元只在确实补位时追加并标明，不伪装成窗口内内容。
- 非选稿内容置底、总分为 0，但不得省略。
- 每条短评为 100–180 个汉字。讲解卡不设机械字数，以能准确支撑约二十五分钟为准；去掉与八段结构无关的背景堆砌。
- 两张讲解卡必须完整，不以“同上”省略任何一段。
- 对初学者保持低概念密度，但不使用会扭曲机制的生活类比。
- 资本关注只作为商业化信号，不输出证券买卖建议、目标价或收益承诺。
- 全文使用中文；技术术语首次出现时保留英文名。
- 只转述外部正文，不粘贴原始 HTML；对标题和文本中的 Markdown 控制字符做转义，只输出 `http` 或 `https` 链接。

## 输出前自检

- [ ] 审查窗口、实际匹配订阅和各层计数明确。
- [ ] 每期早报的新闻块均已解析、合并、标为未解析或解释为非新闻结构块。
- [ ] 橘鸦父早报未被评分，机器之心 SOTA 来源未获得来源加成。
- [ ] 同一规范原文未重复计分；多个不同原文的同一事件已进入同一报道簇。
- [ ] 全部审查单元均进入矩阵与逐条短评。
- [ ] 所有已评分行的六项分数之和等于总分，等级与分档一致；未解析行全部为 `— / N/A`。
- [ ] 非选稿内容为零分且未参与 Top 2。
- [ ] 报道簇、近分互补、新闻级已选键和回溯规则均已应用。
- [ ] Top 2 恰好两条，每张讲解卡八段齐全并有一手证据边界。
- [ ] 每张讲解卡包含完整商业化五问和三个讨论问题。
- [ ] 事实、原文报道、推断和猜测已分开。
- [ ] 本地文件未覆盖旧报告，且与当前对话中的日报正文一致。
