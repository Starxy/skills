# Paper Pass

## 定位

每次只处理一篇可唯一识别的学术论文。默认用中文帮助具备大领域基础、但首次接触该细分方向的读者建立完整心智模型；用户明确指定其他语言时再切换。

让最终结果能够独立回答：论文研究什么、作者为何研究、现有工作卡在哪里、核心思路是什么、真正的新贡献是什么，以及证据支持到什么程度和边界在哪里。

## 开始前门槛

1. 确认论文标题、版本和唯一身份。标题或引用对应多个候选时，列出候选及区分信息并等待确认；输入识别出多篇论文时，立即停止：不得检索、获取或解读任何论文，只输出论文标题并要求用户恰好选择一篇；即使用户明确要求在同一回复中阅读多篇，也必须执行此门槛。
2. 取得可读取的主论文全文后再解释。输入只有标题、摘要或部分正文时，先检索论文官网、开放版本、作者主页或可信仓库；扫描件或文本提取缺失时，尝试替代版本或可靠 OCR。
3. 检查核心方法、实验设置和关键证据是否依赖 Supplementary 或 Appendix；依赖时，将对应补充材料纳入必需全文。
4. 明确当前读取版本。arXiv、会议版或期刊版内容不一致时，只以已声明版本为证据基础。
5. 只有论文身份唯一，且理解主线所需的正文、公式、图表与补充材料全部可读时，才进入阅读。材料仍不完整时，按“停止场景”输出并停止；不要生成摘要级替代稿，也不要用常识补齐。

## 两遍阅读

先判断一个主类型和至多一个辅助类型，然后必须读取 [references/paper-types.md](references/paper-types.md)，只应用所选类型的证据镜头。

**第一遍：建立隐藏证据账本。** 按原文顺序逐节阅读，忽略参考文献、致谢和作者信息等非主体内容。对每个关键观点记录：

- 观点或主张；
- 数字、样本、方法名、比较对象和成立条件等具体细节；
- 它在论证中的作用：动机、缺口、机制、证据、贡献或限制；
- 一段能直接支持该点的原文引文；
- 章节、页码、Figure、Table、Equation、Algorithm 或可识别段落等精确来源锚点。

可合并相近观点，但不要因最终篇幅有限而跳过内部提取。只有每个关键点均有直接引文和来源锚点，且六个认知问题都能从账本找到依据时，第一遍才完成。账本始终是隐藏中间产物，不向用户展示。

**第二遍：沿作者思考链重组。** 仅在账本完成后，按“研究动机 → 现有缺口 → 核心思路 → 实现机制 → 支撑证据 → 贡献与边界”重组材料：

- 将作者明确陈述的动机与解释性重建分开；重建时明确写“从论文的论证结构来看”等归因措辞。
- 将贡献分为核心新意、关键设计、数据或系统资源、验证性结果；不要平铺或等量复述作者的 contribution 列表。
- 只把理解主线所必需的内容写入最终稿，但让每个关键结论都能回溯到账本。

## 最终输出

正常完成时严格使用以下顺序和标题，不展示证据账本：

```markdown
## 核心总结

核心总结须为单段，同时交代研究问题、重要性、核心思路、主要贡献和结论；验收范围为 150–250 个汉字，仅计汉字，不计 Markdown 标记、空白、拉丁字母、数字和标点，以 170–220 个汉字为写作目标；输出前核对实际计数，少于 150 个则扩写，超过 250 个则压缩。

## 论文信息

- 标题：
- 年份 / venue：
- 链接：
- 所属方向：

## 1. 作者为什么要做这项研究

## 2. 现有工作卡在哪里

## 3. 论文的核心思路

## 4. 核心思路如何落地

## 5. 论文如何证明自己的主张

## 6. 贡献与边界
```

`论文信息`必须恰好包含模板列出的四个字段，不得增添其他字段；版本信息按适用情况并入“年份 / venue”或“链接”，不得另设“读取版本”字段；作者、机构、DOI 及其他元数据不得作为独立字段加入。

核心总结和论文信息不计入主体篇幅。根据概念跳跃、必需前置知识、方法模块或论证环节、证据与实验设计的理解成本，在约 1000、2000 或 3000 中文字的软档位中选择；六节主体必须落在 1000–3000 中文字内，并按论文类型动态分配篇幅。未核验的年份、venue、代码或外部元数据标为“未核验”，不要填入默认值。

## 证据与表达

- 始终区分四层：论文明确陈述、原文证据直接支持、基于全文的解释性重建，以及未证明或未覆盖。
- 将关键结论、具体数字和实验结果与紧凑行内锚点绑定，例如“……（§4.2，Table 3）”；论文级链接不能代替主张级锚点。
- 只为定义、核心主张或容易转述失真的内容保留最终直接引文，不倾倒账本中的逐点引文。
- 以读取版本的原文判断论文内容、动机和贡献。外部资料仅用于核验元数据或补足理解主线必需的少量前置概念；给出可点击链接并与论文内容明确区分。
- 默认不展示公式。只有不讲公式就无法理解核心贡献时才引入；先解释关系和作用，再解释出现的每个非常见变量或参数，不做完整推导。
- 若论文含有理解核心机制或主要证据所必需的 Figure 或 Table，最终稿必须恰好选取并解释其中一至两个，且不得单独解释第三个及后续图表；其他 Figure 或 Table 仅可作为紧凑的主张级来源锚点引用，不得另作讲解。若不存在此类必要图表，不要为满足数量而选取无关项。
- 保留必要学术专业性，降低单位段落的概念密度。重要术语首次出现时给出中英对照，后文保持同一译名；不要用不精确的生活类比替代定义。

## 停止场景

停止场景只输出解决当前门槛所需的信息，不套用核心总结、论文信息或六节主体：

- **全文缺失：** 列明已取得材料、缺失材料、尝试过的检索路径，以及无法开始的原因；只请求可读的完整主论文，若理解核心内容依赖补充材料，再一并请求对应的具体补充材料；不要将摘要、节选章节、引言/方法/实验/结果组合、截图或其他部分文本表述为足以开始阅读。然后停止。
- **身份不唯一：** 列出最可能候选及区分信息，等待用户确认。
- **多篇输入：** 列出论文标题，要求用户选定一篇。
- **核心内容不可读：** 指出不可读的章节、图表、公式或补充材料，以及尝试过的替代来源，然后停止。

即使用户同时要求，也不要切换到多篇比较、代码复现、工程接入、完整公式推导、完整同行评审、全文翻译、逐图逐表讲解或阅读建议。只完成本 skill 的单篇首次完整阅读，并简明说明额外任务不在本次范围内。

## 输出前自检

逐项核对；普通输出有未满足项时先修正，门槛项不满足时按停止场景结束：

- [ ] 只处理一篇身份唯一且版本明确的论文。
- [ ] 主论文全文和理解核心所依赖的补充材料全部可读。
- [ ] 已选择一个主类型和至多一个辅助类型，并读取对应类型镜头。
- [ ] 隐藏账本覆盖每个关键点的具体细节、论证作用、直接引文和精确锚点。
- [ ] 作者明确动机与解释性重建已清楚分开。
- [ ] 输出顺序、固定标题、150–250 字核心总结和 1000–3000 字主体均符合契约。
- [ ] 每个关键结论、数字和实验结果都有紧凑行内来源锚点。
- [ ] 贡献已按核心新意、关键设计、资源和验证结果分层，边界说明了证据未证明什么。
- [ ] 公式和图表遵守最小充分原则，内部账本没有泄露。
- [ ] 没有扩展到比较、复现、完整推导、完整审稿、翻译或阅读建议。
