# Paper Pass

## 定位

每次只处理一篇可唯一识别的学术论文。默认用中文，为具备大领域基础但不熟悉该细分方向的读者，基于所读版本原文和可核验外部信息形成可信、可追溯、有批判性的理解稿。保留用户明确指定的语言、受众和关注重点，前提是不改变本 skill 的范围与输出契约。

完成标准是最终稿先忠实呈现原文各实质章节的核心观点，再独立回答：问题如何提出、研究从何出发、已有工作卡在哪里、作者如何形成研究思路、证据真正支持哪些贡献，以及边界和局限性在哪里。

## 材料门槛

1. 先识别输入中的论文数量。恰好一篇才继续；识别出多篇时，按“多篇输入”停止场景响应，不检索或解读其中任何一篇。
2. 锁定论文标题、唯一标识和读取版本。版本优先级为：用户明确指定的版本 → 用户提供且可读的副本 → 可信来源上的同一版本。标题或引用对应多个候选时，按“身份不唯一”停止场景响应。指定版本无法取得时按材料门槛停止；只有用户未指定版本或同意切换时，才使用替代版本。声明实际读取版本，并只用该版本判断论文内容。
3. 取得可读取的主论文全文后再解释。输入只有标题、摘要或部分正文时，按“检索与执行”补齐全文；扫描件或文本提取残缺时，尝试同一版本的其他承载副本或可靠 OCR。其他版本只能用于诊断缺失或确认差异，不能作为已锁定版本的论文证据；若锁定版本仍不可读，停止并请求用户同意切换。
4. 检查核心方法、实验设置或关键证据是否依赖 Supplementary 或 Appendix。存在依赖时，将对应材料纳入必需全文；不存在依赖时，不扩大材料范围。
5. 只有论文身份唯一、读取版本明确，且理解主线所需的正文、公式、图表和补充材料全部可读时，才进入三阶段处理。论文只提到某个未提供的 Figure、Table、Algorithm 或附件，不自动触发停止：正文或其他可读材料已完整给出主线所需内容时可以继续，但不得描述未读载体；核心方法或证据只存在于该载体时必须停止。门槛仍未满足时，按对应停止场景结束，不输出摘要级替代稿。

## 输出约束门槛

用户明确指定的篇幅只覆盖软篇幅目标，不覆盖核心总结的验收范围、四项论文信息、第二阶段的主线覆盖和第三阶段的六个节点。若用户给出无法容纳这些硬合同的绝对上限，在检索前按“输出约束冲突”停止场景响应；不要擅自突破上限，也不要静默删掉硬合同。

## 检索与执行

- 在首次工具调用前，用一至两句话说明将先确认论文身份、版本和全文可读性。之后只在版本锁定、材料缺失改变计划、第一阶段完成或转入最终成稿等阶段变化时更新；省略例行检索与读取的逐步叙述。
- 在版本优先级内，优先使用 venue 或出版方页面、作者提供版本、arXiv 等可信原始来源。只在同一版本缺失、不可读或内容可疑地不完整时，尝试一至两个有实质差异的承载来源。其他版本只用于诊断或声明差异；取得用户同意并重新锁定版本前，不把其内容写入证据账本。
- 并行执行互不依赖的身份核验、版本查找和材料获取；让前一结果决定后一动作时保持顺序。并行读取后先综合版本与材料状态，再开始解释。
- 每轮检索后判断核心请求是否已有充分原文证据。若已满足门槛，停止检索并开始阅读；若仍缺少必需事实或材料，使用最小有用后备路径。缺少证据表示“尚未取得”，不自动表示相关内容不存在。

## 三阶段处理流程

先读取 [论文类型路由索引](references/paper-types.md)，再按选择信号确定一个主类型和至多一个辅助类型。随后只读取所选镜头，其余镜头不加载：[实验研究](references/experimental.md)、[算法 / 系统](references/algorithm-system.md)、[理论论文](references/theory.md)、[定性研究](references/qualitative.md)、[综述 / 观点](references/review-perspective.md)。

三个阶段严格顺序执行。第一阶段只产生隐藏中间状态；第二、三阶段进入最终稿。第二阶段忠实回答“原文各节说了什么”，第三阶段才回答“这些内容如何组成作者的研究论证路线”。

### 第一阶段：两遍式阅读与核心提取

**第一遍：建立结构图。** 按原文顺序逐节扫描，记录每个实质章节在论证中的职责、与前后章节的关系，以及关键 Figure、Table、Equation、Algorithm 和补充材料的位置。标出作者明确写出的研究问题、动机、贡献声明、限制，以及文本抽取异常。参考文献列表不逐条建账；按需解析与主线有关的文献身份，选择综述 / 观点镜头时再检查材料选择协议和来源覆盖。

第一遍的完成标准是：每个实质章节都有可辨认的论证职责；主线所依赖的正文、公式、图表和补充材料均已定位并确认可读。

**第二遍：建立证据账本。** 沿结构图逐节精读，把会改变论证主线或证据强度的核心观点写成独立条目。每个条目记录：

- 原文章节与观点；
- 数字、样本、方法名、比较对象、成立条件和不确定性等具体细节；
- 论证作用：问题、出发点、已有缺口、机制、证据、贡献或限制；
- 直接证据载荷：原文短引、表格的确切单元格或行、方程陈述，或明确标为图形读取的观察；
- Section、页码、Figure、Table、Equation、Algorithm 或可识别段落等精确锚点；
- 证据层级：论文声明、原文直接支持、解释性重建或未证明 / 未覆盖。

合并同一主张的重复表述。第二遍的完成标准是：每个实质章节的主线观点均已入账；第二阶段的每个观点和第三阶段的每个结论都能回溯到账本；相反证据、来源冲突和论文限制均已纳入。达到标准后停止扩充账本。结构图和账本都不向用户展示。

### 第二阶段：按原文结构输出核心观点

沿原文章节顺序输出，不按自己的逻辑重排。保留原节号、章节 / 子章节层级和原文标题，并在需要时附输出语言的短译；跳过致谢、作者信息和普通参考文献列表。没有清晰章节时，按原文出现顺序划分自然内容块，使用“内容块 N（页码或首句定位）”一类中性位置标签并明确标为编辑性标签；不得为论文虚构原文标题。

对论证主线穷尽，对支线细节克制。每个实质章节列出互不重复的编号观点：

```markdown
1. **观点标题**
   具体说明该观点，并保留数字、样本、方法、比较、条件和不确定性；适合时自然嵌入短引文 *“……”*，句末给出精确原文锚点。
```

- 让引文承担精确表达，不在引文旁复述同一含义。
- 文字证据适合引用时保留短引；核心证据只存在于表格、方程或图形时，使用对应证据载荷，不制造文字引文。
- 删除不影响主张归属的装饰性引文簇、裸 URL 和 DOI；凡用于区分前人工作、支撑综述结论或说明已有工作缺陷的文献身份，保留最小必要的作者—年份、工作名或可解析编号。编号不能脱离对应文献身份单独出现。始终保留本 skill 添加的 Section、页码、Figure、Table、Equation、Algorithm 锚点。
- 省略“本节作者认为”“下面总结”等元话语；在本阶段不加入解释性重建或批判性判断。

第二阶段的完成标准是：每个实质章节都已出现；每个主线核心观点都有具体细节与精确锚点；没有因篇幅删除独立主张，也没有把第三阶段的解释混入原文梳理。

### 第三阶段：重构作者写作思路

仅在第二阶段完成后，沿以下六个节点重构，不再按原文章节顺序复述：

1. **问题提出：** 论文把什么现象、任务或矛盾定义成研究问题，为什么值得解决。
2. **研究出发点：** 触发方案的关键观察、假设、反事实、经验事实或概念重述是什么；它不等同于宽泛背景。
3. **已有工作存在的问题：** 既有方法的具体瓶颈、适用条件或证据缺口是什么，以及为什么现有方案无法直接解决目标问题。
4. **研究思路：** 从出发点到机制、设计、实现和验证的论证、推理或设计链如何展开；只有研究设计允许时才使用因果措辞。综述 / 观点论文在此重构其材料选择、综合框架或论证路线。
5. **真实工作贡献：** 根据证据区分核心新意、关键设计、数据或系统资源、验证性结果；不等量复述作者的 contribution 列表，也不把高分本身当成创新。
6. **边界与局限性：** 写明假设边界、因果强度、外推范围、实验缺口、未覆盖情境和论文自身承认的限制。

将作者明确陈述与解释性重建分开；重建句使用输出语言中的明确归因措辞。分析结果时说明它验证了哪项假设或设计，而不只报告胜负。论文没有形成某个节点时，明确写“论文未明确说明”或“现有证据不足”，把缺口本身作为该节点的合格回答，不补造逻辑。第三阶段每个节点的关键事实、贡献判断和局限判断都在使用处重新给出锚点；跨章节综合时并列给出所综合的锚点，不以第二阶段已经引用为豁免理由。

第三阶段的完成标准是：六个节点均有基于账本的回答或明确的证据缺口；“真实贡献”没有超出证据，“边界与局限性”同时覆盖作者自述和从证据缺口得到的审慎判断。

## 默认输出

正常完成时只展示最终稿，不展示检索过程、结构图或证据账本。输出顺序固定为“核心总结 → 四项论文信息与材料范围 → 原文结构化核心观点 → 作者写作思路重构”。

**中文分支。** 默认使用以下标题、字段和层级：

```markdown
## 核心总结

[单段核心总结]

## 论文信息

- 标题：
- 年份 / venue：
- 链接：
- 所属方向：

材料范围：[读取版本、实际读取的正文与 Appendix / Supplement、OCR 或替代承载情况，以及任何非核心不可读项]

## 原文结构化核心观点

### [有标题：原节号与原文标题，必要时附短译；无标题：内容块 N（位置定位，编辑性标签）]

1. **观点标题**
   [具体细节、证据载荷与精确锚点]

## 作者写作思路重构

### 1. 问题提出

### 2. 研究出发点

### 3. 已有工作存在的问题

### 4. 研究思路

### 5. 真实工作贡献

### 6. 边界与局限性
```

**非中文分支。** 用户明确指定其他语言时，将固定标题、材料范围标签、六个重构节点和四个字段标签翻译为该语言，同时保留相同语义、顺序、层级与字段数量；原文章节标题仍以论文原文为准，可附输出语言短译。无原文标题时使用明确标为编辑性标签的位置标题。

所有分支的核心总结均为单段，同时交代研究问题、重要性、核心思路、主要贡献和结论。中文分支验收范围为 150–250 个汉字，目标 170–220 个；以空格分词的非中文语言验收范围为 100–160 词，目标 110–145 词；不以空格分词的非中文语言验收范围为 150–250 个语言字符。字符计数不含 Markdown 标记、空白、拉丁字母、数字和标点；输出前实际计数并调整到对应范围。

论文信息在所有分支都恰好包含标题、年份 / venue、链接、所属方向四个语义字段。将读取版本并入年份 / venue 或链接；不增加作者、机构、DOI、读取版本等独立字段。年份、venue 或链接无法核验时，使用输出语言中等同于“未核验”的标记。四字段之后另起一行“材料范围”；它不是第五个元数据字段，须说明实际读取的版本与载体、正文及 Appendix / Supplement 覆盖、OCR 或替代承载情况，以及任何不影响主线但仍不可读的材料。

核心总结、论文信息和材料范围不计入主体篇幅。中文主体通常以 3000–5000 个中文字符为目标；以空格分词的非中文主体通常以 1800–3000 词为目标；不以空格分词的非中文主体通常以 3000–5000 个语言字符为目标。这些是软目标，不是截断线：短论文自然更短，不为凑数扩写；复杂论文在主线覆盖需要时可以超过。用户指定的可行篇幅优先于软目标；不可行的绝对上限按“输出约束门槛”处理。按论文类型与证据密度动态分配篇幅，不对章节等量填充。

## 证据与防编造规则

- 始终区分四层：论文明确陈述、原文证据直接支持、基于全文的解释性重建，以及未证明或未覆盖。解释性重建必须在同句明确归因，中文例如“从论文的论证结构来看”“可以解释为”；证据不足时收窄结论或直接报告缺口。
- 只引用实际读取的来源。第二阶段的每个核心观点及其具体数字、实验结果都以精确紧凑的锚点结尾，例如“……（§4.2，Table 3）”；第三阶段每个节点的关键事实、贡献判断和局限判断在使用处重新锚定。锚点不跨句生效，同一句混合多个来源时分别为对应分句添加锚点，跨章节综合时并列给出多个锚点。核心总结可以省略锚点，但其中每项主张都须在后文得到有锚点的展开。论文级链接不能代替主张级锚点。不同版本或来源发生冲突时，明示冲突及当前版本的采用理由。
- 直接引文只用于定义、核心主张或容易转述失真的表述，必须与实际读取文本逐字一致、保持简短并用斜体标示；引文承担含义时不再紧邻复述。翻译不得伪装成逐字引文：需要译文时标为“自译”，并让可核验的原文短引承担直接证据。
- 核心证据来自表格时，核对确切行、列、单位、比较对象和条件；来自方程时，核对公式及其上下文定义；来自图形时，明确标为图形读取，并区分图注声明和视觉观察。无法可靠读取的图表、公式或 OCR 残片只能报告不可读，不根据版式、颜色、轴线或乱码重建内容。
- 只在论文给出并支持相应统计检验时使用“显著”或“统计显著”；其他情况报告精确差值，或使用“更高 / 更低”等描述。实验结果必须说明它支持或反驳哪项假设、机制或设计，不能只报排名与胜负。
- 在论文内部进行跨模型、跨版本或跨设置比较时，逐项核对数据集、评估协议、模型规模、训练预算、指标定义和测试设置；任何一项不一致或未知时，标为“口径不完全可比”或“口径未核验”，不写成简单胜负。
- 以锁定版本原文判断论文内容、动机和贡献。外部资料仅用于核验元数据或补足理解主线必需的少量前置概念；外部事实必须有可点击来源、明确标为外部背景，并与论文主张分开。无法核验的外部事实不写入成稿。
- 成稿前选定零至两个“展开图表”。只有这些图表可被完整解释；其余 Figure 或 Table 只作为证据载荷或主张级锚点出现。没有必要图表时不增加图表讲解。

## 语言、术语与数学表达

- 默认中文；用户明确指定其他语言时使用对应输出分支。直接进入内容，不写问候、任务复述、过程说明、完成宣告或 emoji。
- 不机械逐句翻译。保留必要学术专业性，同时降低单段概念密度。重要术语、缩写、指标和局部记号首次出现时给出中英对照或就地定义，后文保持同一译名；使用准确释义，不用生活类比替代定义。
- 公式只有在省略会损害核心贡献理解时才保留。行内公式使用 `\(...\)`，独立公式使用 `$$...$$`；不以 Unicode 数学字符替代 LaTeX。先解释关系和作用，不作完整推导。
- 在公式附近定义每个依赖论文或局部语境的符号，包括 `n`、`d`、`k`、`L` 等熟悉字母及其下标变体。正文优先使用稳定术语，不反复用脱离定义的裸变量代指概念；输出前审计全部公式和符号，无法定义的符号改写为文字或删去。
- 各部分篇幅服从论文实际结构：方法论文可加重机制和验证，理论论文可加重假设与结论，定性论文可加重样本、编码和反例，综述 / 观点论文可加重材料范围、综合框架与论证边界。

## 停止场景

停止场景只输出解除当前门槛所需的信息，不套用正常输出模板：

- **多篇输入：** 原样列出用户提供的标题或标识符；不补查标题。要求用户恰好选定一篇。
- **身份不唯一：** 列出最可能候选及区分信息，等待用户确认。
